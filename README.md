# code-gakko

## コミットルール

### フォーマット

```
<タイプ>: <簡潔な説明>
```

### タイプ一覧

- **feat**: 新しい機能の追加
- **fix**: バグ修正
- **docs**: ドキュメントの変更
- **style**: コードの見た目の変更（フォーマット、インデントなど）
- **refactor**: リファクタリング（新機能追加やバグ修正なし）
- **test**: テストコードの追加・修正
- **chore**: ビルドプロセスやヘルパーテストの追加・変更

### 例

```
feat: ユーザー登録機能を追加
fix: ログイン画面のバリデーションエラーを修正
docs: README.mdを更新
style: コードフォーマットを統一
refactor: ユーザーモデルのリファクタリング
test: ユニットテストを追加
chore: ESLint設定を更新
```
